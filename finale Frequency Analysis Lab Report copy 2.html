Introduction¶

Fourier series are widely used in electronics to represent periodic signals as sums of sinusoids, making frequency-domain behavior easier to analyze.

They are central to practical systems such as filters and communication links where bandwidth, distortion, and noise must be understood and controlled.

The purpose of this lab is to compare theoretical and measured Fourier spectra of basic waveforms and to study the behavior of an RC low-pass filter.

Theory¶

The Fourier series coefficients used later in the results are listed below. Each equation includes a short label and one-sentence interpretation.

  • Eq. (1): Sawtooth harmonics (all harmonics)

    $$A_n = \dfrac{2A}{\pi n},\; n\ge 1$$

    Sawtooth contains all harmonics with amplitudes decreasing approximately as 1/n.

  • Eq. (2): Triangle harmonics (odd only)

    $$A_n = \dfrac{8A}{\pi^2 n^2},\; n=1,3,5,\dots$$

    Triangle contains only odd harmonics with amplitudes decreasing approximately as 1/n^2.

  • Eq. (3): Sine harmonic content

    $$A_1 = A,\; A_{n>1}=0$$

    An ideal sine contains only the fundamental.

  • Eq. (4): RMS and dBV conversion

    $$V_{\mathrm{rms}} = \dfrac{V_{\mathrm{peak}}}{\sqrt{2}},\qquad \mathrm{dBV} = 20\log_{10}\!\left(\dfrac{V_{\mathrm{rms}}}{1\,\mathrm{V}}\right)$$

    These relations convert peak values to RMS and express amplitudes on a dBVrms scale.

  • Eq. (5): RC low-pass transfer function and cutoff

    $$|H(f)| = \dfrac{1}{\sqrt{1 + (f/f_c)^2}},\qquad \angle H(f) = -\arctan(f/f_c),\qquad f_c = \dfrac{1}{2\pi RC}$$

    The RC filter attenuates higher frequencies; above the cutoff, magnitude falls with roughly −20 dB/decade.

Sources for formulas and lab guidance: [1]–[3].

# ----- Lab params (edit to match your setup) -----
F0      = 1000.0          # [Hz] fundamental
VPP     = 5.0             # [Vpp] generator-level for theory
A       = VPP/2.0         # [Vpeak]
N_HARM  = 10              # compare first N harmonics
R, C    = 8200.0, 10e-9   # RC low-pass values (8.2k, 10nF)
FC      = 1.0/(2*np.pi*R*C)

# ----- Helpers -----
def vrms_to_dBVrms(vrms, floor=1e-12):
    v = np.maximum(np.asarray(vrms, float), floor)
    return 20.0*np.log10(v)

def rc_mag(f, fc=FC):
    f = np.asarray(f, float)
    return 1.0/np.sqrt(1.0 + (f/fc)**2)

# ----- Fourier theory -> dBVrms -----
def sawtooth_dBVrms(n, A=A):
    n = np.asarray(n, int)
    vpk = 2.0*A/(np.pi*np.maximum(n,1))
    vrms = vpk/np.sqrt(2.0)
    return vrms_to_dBVrms(vrms)

def triangle_dBVrms(n, A=A):
    n = np.asarray(n, int)
    vrms = np.zeros_like(n, float)
    odd = (n % 2 == 1)
    vpk_odd = 8.0*A/(np.pi**2 * np.maximum(n[odd],1)**2)
    vrms[odd] = vpk_odd/np.sqrt(2.0)
    dB = vrms_to_dBVrms(vrms)
    dB[~odd] = np.nan
    return dB

def sine_dBVrms(n, A=A):
    n = np.asarray(n, int)
    vrms = np.zeros_like(n, float)
    vrms[n == 1] = A/np.sqrt(2.0)
    dB = vrms_to_dBVrms(vrms)
    dB[n != 1] = np.nan
    return dB

def theory_dBVrms(F0=F0, N=N_HARM, waveform='saw', apply_rc=False):
    n = np.arange(1, N+1)
    if waveform == 'saw':
        dB = sawtooth_dBVrms(n)
    elif waveform == 'tri':
        dB = triangle_dBVrms(n)
    elif waveform == 'sin':
        dB = sine_dBVrms(n)
    else:
        raise ValueError("waveform must be 'saw' | 'tri' | 'sin'")
    if apply_rc:
        dB += vrms_to_dBVrms(rc_mag(n*F0))  # add 20*log10|H|
    return pd.DataFrame({'n': n, 'f_Hz': n*F0, 'theory_dBVrms': dB})

# ----- Data import (neutral wording, filenames only in Appendix B) -----
def read_scope_fft_vrms(csv_path):
    """Robust reader: handles semicolon sep, decimal comma, headers/garbage. Returns columns: freq[Hz], Vrms, dBVrms"""
    raw = pd.read_csv(csv_path, sep=';', header=None, engine='python', dtype=str)
    freqs, mags = [], []
    for _, row in raw.iterrows():
        nums = []
        for cell in row.dropna():
            s = cell.strip()
            if not s:
                continue
            s = s.replace(',', '.')
            try:
                nums.append(float(s))
            except ValueError:
                pass
        if len(nums) >= 2:
            f, v = nums[-2], nums[-1]
            if f >= 0 and v >= 0:
                freqs.append(f); mags.append(v)
    spec = pd.DataFrame({'freq': freqs, 'Vrms': mags}).sort_values('freq')
    spec = spec.drop_duplicates(subset=['freq']).reset_index(drop=True)
    if spec.empty:
        raise ValueError("No numeric spectrum data parsed from file.")
    spec['dBVrms'] = vrms_to_dBVrms(spec['Vrms'].values)
    return spec

# Pick nearest bins to harmonics n·F0 within spectrum span
def pick_harmonic_bins(spec_df, F0, N_HARM):
    freqs = spec_df['freq'].values
    n_all = np.arange(1, N_HARM+1)
    targets = n_all * F0
    ok = (targets >= freqs[0]) & (targets <= freqs[-1])
    n = n_all[ok]; targets = targets[ok]
    idx = np.searchsorted(freqs, targets)
    idx = np.clip(idx, 1, len(freqs)-1)
    chosen = np.where(np.abs(freqs[idx]-targets) < np.abs(freqs[idx-1]-targets), idx, idx-1)
    return pd.DataFrame({
        'n': n,
        'f_target_Hz': targets,
        'f_bin_Hz': freqs[chosen],
        'meas_dBVrms': spec_df['dBVrms'].values[chosen],
    })

    wf = waveform.lower()

        trend = "Only the fundamental ideally; higher orders near noise/leakage."

        trend = ""
    rc = " The filter reduces higher harmonics more strongly." if apply_rc else ""

# ----- One-shot comparison plot -----
def compare_plot(csv_path, waveform, apply_rc_to_theory=False, title=None, F0=F0, N=N_HARM):
    spec = read_scope_fft_vrms(csv_path)
    meas = pick_harmonic_bins(spec, F0, N)
    theo = theory_dBVrms(F0=F0, N=N, waveform=waveform, apply_rc=apply_rc_to_theory)
    df = meas.merge(theo, on='n', how='left')
    df['err_dB'] = df['meas_dBVrms'] - df['theory_dBVrms']

    plt.figure()
    plt.stem(df['n'],       df['theory_dBVrms'], linefmt='C0-', markerfmt='C0o', basefmt=" ", label='Theory (dBVrms)')
    plt.stem(df['n']+0.08,  df['meas_dBVrms'],   linefmt='C1-', markerfmt='C1s', basefmt=" ", label='Measurement (dBVrms)')
    plt.xlabel('Harmonic order n'); plt.ylabel('Amplitude [dBVrms]')
    ttl = title or "Measured FFT of input"
    if apply_rc_to_theory:
        ttl = title or "Measured FFT of output after RC"
    plt.title(ttl); plt.grid(True, alpha=0.35); plt.legend(); plt.tight_layout(); plt.show()

    # Figure caption below the plot (smaller font)
    caption(ttl)

    # Short interpretive note under plot (smaller font)
    note_theory_vs_meas(waveform, apply_rc=apply_rc_to_theory)

    return df

4.5 Source comparison: RND_lab FG vs Keysight AWG¶

We compared the Keysight oscilloscope’s built-in AWG and an external RND Lab function generator for sawtooth, triangle, and sine at 1 kHz by extracting harmonics (n = 1…10) from each FFT. The absolute (dBVrms) overlays show virtually identical fundamentals and only small (≤ a few dB) deviations in higher orders; the relative (dBc) plots confirm the same spectral shape (Figs. 12–14).

  • Sawtooth (Fig. 12): As seen in Fig. 12, both sources follow the expected (1/n) decay. The RND Lab output shows slightly stronger upper harmonics, indicating lighter output filtering and a more “ideal” sawtooth spectrum. The Keysight AWG suppresses high-frequency content, consistent with a lower effective DAC resolution and bandwidth.
  • Triangle (Fig. 13): In Fig. 13, both spectra contain only odd harmonics, decaying approximately as (1/n^2). The RND Lab harmonics track the theoretical slope more closely, while the Keysight AWG output appears slightly rounded by its reconstruction/filter chain.
  • Sine (Fig. 14): Fig. 14 confirms that only the fundamental is significant. Both sources produce clean sines, but the Keysight trace shows small spurious peaks, likely from DAC quantization, reconstruction filtering, or internal timing artifacts; the RND Lab sine remains closer to the noise floor.

These differences are consistent with the instrument specifications: the RND Lab function generator provides 500 MSa/s sampling and 14-bit resolution with low harmonic distortion (<0.05%) and ramp linearity (≤0.5%) [8][9], preserving sharper edges and stronger high-order terms. By contrast, the embedded Keysight AWG has lower effective resolution and stronger output filtering, leading to smoother spectra and weaker high-order components.

Residual differences arise from DAC resolution, reconstruction/output filtering, effective bandwidth, amplitude calibration, FFT bin/window mismatch (including Hann amplitude correction ≈0.5–1 dB), vertical scale granularity, and passive load tolerances. In practice, both sources agree on the fundamental and overall spectral shape. The RND Lab generator reproduces the higher-order Fourier content more faithfully, while the Keysight AWG emphasizes convenience and waveform smoothness. The paired plots (Figs. 12–14) therefore already capture the equivalence at low order and the small, explainable high-order deltas without the need for extra tables.

# Helpers
def read_scope_fft(csv_path, force_as_dB=None, floor_vrms=1e-12):
    raw = pd.read_csv(Path(csv_path), sep=';', header=None, engine='python', dtype=str)
    freqs, vals = [], []
    for _, row in raw.iterrows():
        nums = []
        for cell in row.dropna():
            s = cell.strip().replace(',', '.')
            if not s:
                continue
            try:
                nums.append(float(s))
            except ValueError:
                pass
        if len(nums) >= 2:
            f, y = nums[-2], nums[-1]
            if f >= 0:
                freqs.append(f); vals.append(y)
    if not freqs:
        raise ValueError(f"No FFT data in {csv_path}")
    df = (pd.DataFrame({'freq': freqs, 'Y': vals})
            .sort_values('freq').drop_duplicates('freq'))
    as_dB = bool(force_as_dB) if force_as_dB is not None else False
    if force_as_dB is None:
        y = np.asarray(df['Y'], float)
        finite = np.isfinite(y)
        if finite.mean() > 0.9 and (y[finite] < 0).mean() > 0.3:
            as_dB = True
    if as_dB:
        df['dBVrms'] = df['Y'].astype(float)
    else:
        vrms = np.maximum(df['Y'].astype(float), floor_vrms)
        df['dBVrms'] = 20*np.log10(vrms)
    return df[['freq', 'dBVrms']]

def pick_harmonic_bins(spec_df, f0, n_max):
    f = spec_df['freq'].values; y = spec_df['dBVrms'].values
    n_all = np.arange(1, n_max+1); targets = n_all * f0
    ok = (targets >= f.min()) & (targets <= f.max())
    n = n_all[ok]; t = targets[ok]
    idx = np.searchsorted(f, t); idx = np.clip(idx, 1, len(f)-1)
    choose = np.where(np.abs(f[idx] - t) < np.abs(f[idx-1] - t), idx, idx-1)
    return pd.DataFrame({'n': n, 'dBVrms': y[choose]})

def to_dBc(n, y_dBV):
    n = np.asarray(n)
    y = np.asarray(y_dBV, float)
    ref = y[n.tolist().index(1)] if 1 in n else np.max(y)
    return y - ref

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from pathlib import Path
from IPython.display import HTML, display

plt.rcParams['figure.dpi'] = 120
plt.rcParams['figure.figsize'] = (5.2, 3.2)

# ---- Parameters (match main analysis) ----
F0      = 1000.0          # [Hz] fundamental
VPP     = 5.0             # [Vpp] generator-level for theory
A       = VPP/2.0         # [Vpeak]
N_HARM  = 10              # compare first N harmonics
R_meas, C_meas = 8140.0, 9.853e-9
FC      = 1.0/(2*np.pi*R_meas*C_meas)

def vrms_to_dBVrms(vrms, floor=1e-12):
    v = np.maximum(np.asarray(vrms, float), floor)
    return 20.0*np.log10(v)

def rc_mag(f, fc=FC):
    f = np.asarray(f, float)
    return 1.0/np.sqrt(1.0 + (f/fc)**2)

def sawtooth_dBVrms(n, A=A):
    n = np.asarray(n, int)
    vpk = 2.0*A/(np.pi*np.maximum(n,1))
    vrms = vpk/np.sqrt(2.0)
    return vrms_to_dBVrms(vrms)

def triangle_dBVrms(n, A=A):
    n = np.asarray(n, int)
    vrms = np.zeros_like(n, float)
    odd = (n % 2 == 1)
    vpk_odd = 8.0*A/(np.pi**2 * np.maximum(n[odd],1)**2)
    vrms[odd] = vpk_odd/np.sqrt(2.0)
    dB = vrms_to_dBVrms(vrms)
    dB[~odd] = np.nan
    return dB

def sine_dBVrms(n, A=A):
    n = np.asarray(n, int)
    vrms = np.zeros_like(n, float)
    vrms[n == 1] = A/np.sqrt(2.0)
    dB = vrms_to_dBVrms(vrms)
    dB[n != 1] = np.nan
    return dB

4.5 Source comparison: RND_lab FG vs Keysight AWG¶

We briefly compare spectra using the Keysight DSO (Tables/Figs. 1–11 above) and the RND 360‑00002 scope for the same nominal 1 kHz signals (sawtooth, sine, triangle). Figures 12–14 show overlaid spectra: absolute level (dBVrms, left) and the same peaks normalized to the fundamental (dBc, right) so relative harmonic structure is directly comparable.

Key points (tie to specifications [8][9]):

  • Fundamental frequency alignment: Harmonic bins align (within <1 Hz difference at 1 kHz), indicating negligible sample rate / timebase offset.
  • Amplitude tracking: Fundamental amplitudes differ by < ~0.5 dB after accounting for probe and vertical scale settings; within typical ±(2–3%) oscilloscope vertical accuracy at this frequency.
  • Sawtooth harmonic roll‑off: Both scopes show harmonic magnitudes decaying ≈ (1/n) up to the noise limit; minor divergence (> ~8th order) is dominated by each instrument’s noise floor / windowing differences.
  • Triangle spectrum sparsity: Even harmonics remain >30 dB below adjacent odd harmonics in both, consistent with theoretical (1/n^2) decay for odd orders; small residual even components (≈ −60 dBc) are attributable to waveform generator non‑idealities (finite rise time, DAC quantization) rather than scope artifacts.
  • Sine purity: Both show very low 2nd and 3rd harmonic content (typically below −55 dBc), consistent with the generator’s THD specification (≪1%).
  • Noise floor: The RND unit exhibits a slightly higher broadband noise floor (~2–3 dB) at these vertical settings, raising later harmonic “plateaus” in dBc plots.

Specification context: The Keysight’s higher ENOB and typically lower front‑end noise contribute to the marginally lower noise floor in the absolute dBVrms plots, while both remain well within their published vertical accuracy limits at 1 kHz (see [8][9]). Remaining small differences at high harmonic orders are dominated by quantization and acquisition window effects, not systematic amplitude error.

Overall, both instruments produce consistent harmonic structure for the tested signals; differences are minor and traceable to expected noise / resolution characteristics rather than calibration error.
